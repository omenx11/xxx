# ВКР: прогнозирование социально-экономического развития регионов РФ

**Тема ВКР:** Разработка алгоритмов машинного обучения для прогнозирования социально-экономического развития.

**Объект исследования:** социально-экономическое развитие субъектов Российской Федерации.  
**Предмет исследования:** методы и алгоритмы прогнозирования ВРП на душу населения на основе региональных и макроэкономических показателей.  
**Период:** 2001–2023 гг.  
**Регионы:** 85 субъектов РФ.  
**Основная целевая переменная:** ВРП на душу населения в текущих ценах.  
**Дополнительная целевая переменная:** ВРП на душу населения в ценах 2015 года.

Проект предназначен для выпускной квалификационной работы бакалавра. Акцент сделан не на декларации о превосходстве машинного обучения, а на воспроизводимой методологии: подготовка данных → построение лаговых признаков → отбор факторов → честная walk-forward-валидация → сравнение классических и ML-моделей → интерпретация → сценарный прогноз → прототип аналитической системы.

---

## Что исправлено в текущей версии

1. Добавлен **CAGR-бенчмарк** как классический инерционный метод среднегодового темпа роста.
2. Добавлена **ARIMA(1, 1, 0)** по каждому региону как классическая модель временных рядов.
3. Убрана нестабильная нейросетевая модель **MLPRegressor**: для коротких региональных рядов 2001–2023 она ухудшала качество и была методологически слабой для бакалаврской ВКР.
4. Убрана обычная **LinearRegression** без регуляризации. В проекте оставлены Ridge, Lasso и ElasticNet как более устойчивые регуляризованные линейные модели.
5. Исправлена логика финального обучения: теперь отдельно сохраняются модели для номинального и реального ВРП.
6. Исправлен сценарный прогноз: реальный ВРП теперь может прогнозироваться отдельной ML-моделью, а не только индексной постобработкой номинального прогноза.
7. Уточнено описание сценариев: макропараметры — это фиксированные сценарные предположения, а не линейная интерполяция.
8. Уточнена версия Python: минимально Python 3.11, рекомендуется Python 3.12.

## Быстрый старт

### 1. Создание окружения

```powershell
py -3.12 -m venv C:\venvs\grp_vkr
C:\venvs\grp_vkr\Scripts\Activate.ps1
pip install -r requirements.txt
```

Минимальная версия Python — **3.11**, рекомендуемая — **3.12**.

### 2. Полный запуск проекта

```powershell
python scripts/run_pipeline.py
```

Пайплайн выполняет 9 шагов:

1. `Preprocess` — очистка и преобразование данных Росстата.
2. `Feature engineering` — лаги, rolling-признаки, темпы роста, макрофакторы.
3. `Feature selection` — корреляция, VIF, LightGBM importance, permutation importance.
4. `GridSearch` — подбор гиперпараметров на TimeSeriesSplit.
5. `Walk-forward CV` — оценка моделей на 2020–2023 гг.
6. `Train final` — обучение финальных моделей для nominal и real.
7. `SHAP` — интерпретация факторов.
8. `Forecast` — сценарный прогноз 2024–2028 гг.
9. `Visualize` — графики, таблицы и интерактивные карты.

Запуск одного шага:

```powershell
python scripts/run_pipeline.py --step 5
```

Запуск с определённого шага:

```powershell
python scripts/run_pipeline.py --from 5
```

### 3. Генерация ноутбука

```powershell
python scripts/make_notebook.py
jupyter notebook notebooks_vkr/01_vkr_main.ipynb
```

### 4. Запуск прототипа аналитической системы

```powershell
streamlit run app_vkr/streamlit_app.py
```

## Методология

### Целевая переменная

Основной таргет — номинальный ВРП на душу населения:

```text
Y477110006 — ВРП на душу населения, руб.
```

Дополнительный таргет — реальный ВРП на душу населения в ценах 2015 года:

```text
grp_real_pc_2015
```

Реальный ВРП рассчитывается через индекс физического объёма ВРП Росстата:

```text
real_factor(i, 2015) = 1.0
real_factor(i, t) = real_factor(i, t−1) * IFO(i, t) / 100,  t > 2015
real_factor(i, t) = real_factor(i, t+1) / (IFO(i, t+1) / 100),  t < 2015
real_GRP_pc(i, t) = nominal_GRP_pc(i, 2015) * real_factor(i, t)
```

При отсутствии ИФО используется fallback на региональный ИПЦ.

### Контроль утечек

В проекте запрещено использовать текущие значения целевой переменной и её прямые производные как признаки. Все основные социально-экономические показатели включаются только через лаги.

| Источник риска | Решение |
|---|---|
| Целевая переменная текущего года | исключается из `X` |
| ВРП всего региона текущего года | исключается из `X` |
| Региональные индикаторы | используются как `lag1`, `lag2`, `yoy_lag1` |
| Макрофакторы | используются как `lag1` |
| Отбор признаков | проводится до тестового периода |
| Проверка качества | walk-forward на 2020–2023 гг. |

### Walk-forward CV

```text
test_year = 2020: train 2001–2019 → test 2020
test_year = 2021: train 2001–2020 → test 2021
test_year = 2022: train 2001–2021 → test 2022
test_year = 2023: train 2001–2022 → test 2023
```

Такой протокол важен для ВКР, потому что тестовый период включает COVID-2020 и санкционный шок 2022 года.

## Модели

### Классические и базовые модели

| Модель | Назначение |
|---|---|
| Naive | прогноз равен значению прошлого года |
| CAGR | прогноз через среднегодовой темп роста региона |
| MeanGrowth | прогноз через средний годовой прирост региона |
| ARIMA | одномерная ARIMA(1, 1, 0) по каждому региону |

### ML-модели

| Группа | Модели |
|---|---|
| Регуляризованные линейные модели | Ridge, Lasso, ElasticNet |
| Деревья и ансамбли | RandomForest, GradientBoosting |
| Бустинг | XGBoost, LightGBM, CatBoost при наличии |

ML-модели обучаются на `log1p(target)`, прогноз возвращается через `expm1`.

## Корректная интерпретация результатов

Для ВРП на душу населения характерна высокая инерционность. Поэтому сильный результат CAGR, MeanGrowth или Naive не является недостатком исследования. Напротив, это важный вывод: региональные траектории ВРП устойчивы, а ML-модели должны оцениваться не только по точности, но и по способности учитывать дополнительные факторы и давать интерпретируемый сценарный анализ.

В тексте ВКР следует избегать формулировки «ML однозначно лучше классических методов». Корректная формулировка:

> В работе проведено сравнение инерционных, классических временных и ML-моделей. Результаты walk-forward-валидации показывают, что ВРП на душу населения обладает выраженной инерционностью, поэтому простые модели роста являются сильными бенчмарками. ML-модели используются для учёта нелинейных зависимостей, интерпретации факторов и построения сценарного прогноза.

## Практический результат

Практическая часть ВКР включает прототип аналитической системы на Streamlit. Система позволяет:

- просматривать динамику ВРП регионов;
- сравнивать регионы и федеральные округа;
- анализировать прогноз по одному региону;
- сравнивать baseline, optimistic и pessimistic сценарии;
- строить интерактивную карту регионов РФ;
- просматривать SHAP-интерпретацию модели.
